## 📌 What I Learned Today

### Understanding the For Loop

A `for` loop lets you **repeat a block of code** a specific number of times without writing it again and again. Instead of writing the same line 10 times, you write it once and tell the loop how many times to run it.

It has 3 parts in one line — a starting point, a condition to keep running, and a step to move forward each time. The loop keeps going as long as the condition is true, and stops the moment it becomes false.

---

### How it flows

- **Initialization** — where do we start? (e.g. start from 1)
- **Condition** — when do we stop? (e.g. stop when we reach 10)
- **Update** — how do we move forward? (e.g. add 1 each time)

## 💻 Programs I Built Today

### 1. Fibonacci Series
The Fibonacci series is a sequence where every number is the **sum of the two numbers before it** — starting from 0 and 1. So it goes: 0, 1, 1, 2, 3, 5, 8, 13...

I used a for loop to keep adding the previous two numbers and printing the next one in the series up to a given count.

---

### 2. Prime Number Checker
A prime number is a number **greater than 1 that has no divisors other than 1 and itself** — like 2, 3, 5, 7, 11.

I used a for loop to check every number from 2 up to the given number and see if it divides evenly. If nothing divides it, it's prime.

---

### 3. Factorial
The factorial of a number is the **product of all whole numbers from 1 up to that number**. For example, factorial of 5 = 5 × 4 × 3 × 2 × 1 = 120.

I used a for loop to multiply numbers one by one from 1 to the given number and keep a running total.

---

### 4. Sum of Natural Numbers
Natural numbers are simply 1, 2, 3, 4, 5... The task was to **add all numbers from 1 to N**.

The for loop went through each number one by one and kept adding it to a total — giving the final sum at the end.

---

### 5. Multiplication Table
Used a for loop to print the multiplication table of any number the user enters — from 1 to 10.

Simple but a great way to see how loops repeat with a changing value each time.

---

## 🧠 Key Takeaways

- A for loop is used when you **know how many times** you want to repeat something
- The 3 parts — start, condition, update — all work together to control the loop
- Loops are at the heart of almost every DSA problem you will ever solve
- Fibonacci, factorial, and prime checking are **classic problems** that appear in interviews
- Using a loop instead of writing repeated code makes your program shorter, smarter, and easier to read

---

## ⚡ When to use a For Loop

| Situation | Use For Loop? |
|---|---|
| Print numbers 1 to 100 | ✅ Yes |
| Repeat something unknown number of times | ❌ Use while loop |
| Go through each item in a list | ✅ Yes |
| Run something forever | ❌ Use while loop |
